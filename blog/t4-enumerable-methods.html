THE #MAP METHOD

2.28.2015

WHAT IS #MAP?

The method #map is built into the Enumerable module of Ruby. Its main function is to:

1. Traverse (move across) the array from element to element.
2. Manipulate the elements one by one via a code block.
3. Produce an entirely new array of the manipulated objects.

As you can see from the points above, #map is a similar method to #each as it traverses over elements inside an array, but with a few key distinctions. The #each method iterates over an array, but does not change it in any way, even if passed a code block.

array = [10, 20, 30]
array.each {|i| i * 2}
=>[10, 20, 30]

As you can see from the example, #each returned the original elements from the array and didn't change them at all. So then, how do we iterate over this array and multiply each element by 2? This is where #map comes into play. With #map we can go through the array, transform each element, and create a new array with the result.

array = [10, 20, 30]
array.map {|i| i * 2}
=>[20, 40, 60]

Sucess! Through using #map we were able to take the element 10, multiply it by 2 and make it 20! We took 20 and made it 40, and so on.

Now we know how to change the elements of an array with the #map method, but the fun doesn't stop there. As I mentioned above, #map creates an entirely new array meaning that we can store these new values into a variable without overwritting our original array!! How cool is that?! Let's give it a shot.

array = [10, 20, 30]
times_two = array.map {|i| i * 2}
puts times_two
=>[20, 40, 60]
puts array
=>[10, 20, 30]

Looking closely, we can see that calling the #map variable "times_two" gives us the new array with modified elements. When we call the original array "array" it prints out the original element values. However, let's say you do in fact want to change the elements of the original array, we can do that too by using the method #map!. This will change "array" in the example above to print the modified values.

There you have it! The in's and out's of the #map method! It's definitely a handy card to have up your ruby sleave when manipulating data structures.
