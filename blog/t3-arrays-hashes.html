When to Array and When to Hash

2.19.15

So there you are, working on your program, when all of sudden you realize that you need a structure to store data! What do you turn to? Look no further than to your friendly Array or Hash. This blog post will cover what these data storing object indexers are, how they work and how to recognize which structure to use.

Arrays and Hashes are index’s of collected objects. In other words, they store objects inside themselves in an organized fashion so that these objects can be called on by the program. Think of these structures as linked set of treasure chests each with an object inside it. When the program wants something inside one of the chests, it presents a key to open the chest and access the object.

ARRAYS

Arrays are unique in that they only use an ordered set of integers as keys. In Arrays, the integer count starts at 0 instead of 1. This can take some getting used to. For example, say we wanted to access the number "2" in the following array: Numbers = [1, 2, 3], we would do so by calling the array followed by the index: Numbers[1], which would unlock the chest at first index and give us the number "2".

Arrays can be created in a couple different ways. One way is to use brackets, like so:

first_three_months = [“Jan”, “Feb”, “Mar”]

Shortcut Alert! By using the %w notation, we can write out our strings without commas or quotes :)

first_three_months = %w[Jan Feb Mar]

We just created an array called first_three_months! Awesome!

Now the second way to create an array is by using the .new method. For example:

first_three_months = Array.new

From there we can index objects into the array:

first_three_months[0] = “Jan”
first_three_months[1] = “Feb”
first_three_months[2] = “Mar”

Excellent! You’re on your way to becoming an Array champion! Now let’s see what’s going on in the world of Hashes.

HASHES

Like Arrays, Hashes are structures that index objects, however instead of using an ordered enumerable system, Hashes use key value pairs. Nothing to it! Instead of one object in brackets like we did in Arrays, we’ll have two objects (the key and the value) in braces for Hashes. Let’s create a hash to store a username, password and email.

user_info = {
username: sean,
password: 1234,
email: sean@sean.com
}

Boom! There it is. And just like Arrays, you can use the .new method to create a Hash as well:

user_info = Hash.new

Hashes are also indexed using the block notation.

user_info[username:] = sean

Unlike Arrays however, Hashes store a default value of “nil” when a key is called that doesn’t have a value.

user_info[age:] = nil

WHEN TO USE WHAT

Now that we have a basic understanding of what Arrays and Hashes do, let’s quickly talk about when to use them.

Array’s are rigid, but efficient. In most cases you’d want to use an array when the objects follow a numeric sequence that aligns with the index numbering, such as the first_three_months example.

Hashes are more flexible. They are best used when a numeric index doesn’t match well with the objects being stored, such as the user_info example. It’s much easier to call the value “sean” by referencing the username: key than by referencing index 0.

And there you have it! Arrays and Hashes are yours for the taking. Use them wisely!
