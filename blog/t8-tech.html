REGULAR EXPRESSIONS IN RUBY

3.27.15

One topic that wasn’t covered in Phase 0 of DBC, but had me insanely curious, was that of regular expressions in Ruby. Through out Phase 0 I’d hear my cohort discuss Regular Expressions (they’d call it Reg Ex for short) at length and I’d have no idea what they were talking about. So for this topic, I’d like to cover the basics. What is a regular expression? Why is it useful? Let’s dive in!

What is a regular expression?

A regular expression is a pattern of characters to be matched in a string. In Ruby, you can create a regular expression by writing a pattern between slash characters (/pattern/) followed by =~ operator. Let’s take a look at an example 

  /start/ =~ “Can’t wait to start DBC!”

  => 14

Here we’re making the regular expression /start/ and asking Ruby if the string contains it, and if so, where does it start in the string. In this case, “start” starts on the 14th index of the string.

We can also use regular expressions to return boolean values using the !~ (does not have) operator. Let’s see if “SF” is in our string. 

  /SF/ !~ “Can’t wait to start DBC!”

  => true

“SF” is not included in our string value, so Ruby evaluates the expression to true.

Using the =~ operator is cool, but Ruby also has a method .match that is even more powerful when using regular expressions. Let’s check it out. 

  my_string = “Can’t wait to start DBC!”

  reg_exp = /start/.match(my_string)

  => #

Here we can see that the match method returns the characters that matched, and we’ve only scratched the surface. Watch this. 

  reg_exp.match.to_s => “start”

  reg_exp.pre_match => “can't wait to "

  reg_exp.post_match => " DBC!"

As you can see, we were able to return the matched value, the values before the match AND the values after the match using different match methods. Awesomness, I know.

Well there you have it! A small look into the world of regular expressions.
