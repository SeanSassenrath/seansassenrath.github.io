CSS - MARGIN vs BORDER vs PADDING

2.15.15

After an intense week studying HTML and CSS, I've fallen in love with the spaceing options available through the margin, border and padding properties in CSS.

The key to great design is spacing. Great spacing allows you to clearly section off ideas with text and imagery. If we had multiple ideas wrapped up next to each other, it would create a very frustrating experience for the user. Thankfully, we have the margin property in CSS! These property allows us to push our HTML elements away or towards other HTML elements till we achieve the correct amount of space.

Another important aspect to design is to see where the margin is being created from - a guide of sorts. This is where borders come into play. By using the border property in CSS we are able to see the boundry of the HTML element and make our other style property adjustments accordingly. It also creates a great look if you're going for a colored border theme on buttons or images. We can also manipulate the border from the default box with right angles to a more smooth rounded corner design by using "border-radius".

Lastly, there is the space inside the HTML element itself. This is called padding. It's the space that encapsules the content and pushes up against the border and margin. Often padding is used when link buttons are created (like the ones on this site!).

And there you have it! The uses of margin, border and padding! I'd recommend checking out this article by SmashingMagazine.
