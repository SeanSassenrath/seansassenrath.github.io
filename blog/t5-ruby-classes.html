WORKING OUT WITH RUBY CLASSES

Coding design and architecture is crucial when it comes to OOP (Object Oriented Programming). It’s important to build code that is organized, easy to read and flexible to updates. Fortunately, Ruby has a great tool for helping us maintain an organized structure, this tool is called a class. A class is a blueprint for creating individual objects with certain characteristics and functions. It’s best that a class is used for one single responsibility or purpose. For this post lets create a class called “GymWeight”.

class GymWeight
end

There we have it! Our class has been created. Since this is a blue print for any number of different types of weights, let’s begin filling it in with some characteristics.

class GymWeight
--def initialize(weight, material)
----@weight = weight
----@amount = material
--end
end

Now we have some characteristics of the class GymWeight by calling on a special method called “initialize”. This is a method that “creates” the class and allows us to pass arguments through the class (weight and material) for our class to use. In order to use these arguments in different methods of our class, we assign them instance variables. These special variables store the arguments as usable objects throughout our program (ie we can use them in different methods).

class GymWeight
--def initialize(weight, material)
----@weight = weight
----@material = material
--end
--def weight_amount
----if @weight > 100
------puts "You can lift #{@weight}lbs?! That’s heavy."
----else
------puts "You can lift {@weight}lbs? Not bad."
----end
--end
--def material_type
----puts "My favorite type of weight is made out of #{@material}."
--end
end

GoldsGym = GymWeight.new(150, "metal")
GoldsGym.weight_amount
GoldsGym.material_type

There looks like a lot here I know, but let’s take it piece by piece. First, we created two new methods “weight_amount” and “material_type”. We then feed them the arguments that were passed to the GymWeight class when we create the object “GoldsGym”. In this case, it was “150” and “metal”. We could then call on the method inside the our class on the GoldsGym object to return the values of the program.

That’s all there is to it! Not the most amazing example I know, but hopefully it gives you a sense of how classes work and when to use them.
